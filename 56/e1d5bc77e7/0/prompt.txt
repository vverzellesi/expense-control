# Committing Changes

Create git commits for changes made during this session.

## Process

1. **Analyze changes:**
   - Run `git status` and `git diff` to understand modifications
   - Consider whether changes should be one or multiple logical commits

2. **Plan commits:**
   - Group related files together
   - Draft clear commit messages in imperative mood
   - Focus on why changes were made

3. **Execute upon confirmation:**
   - Use `git add` with specific files (never `-A` or `.`)
   - Create commits with `git commit -n -m "message"`
   - Show result with `git log --oneline -n [N]`

## Critical Rules

- **NEVER add Claude attribution** - no co-author lines, no "Generated with Claude"
- Write commit messages as if the user wrote them
- Keep commits atomic and focused
